\section{Overview}

This is a quick showcase of the best (read: flashiest) LaTeX features. Its two main pulls (generally) are dynamic control and vectorisation. 
A LaTeX project is broadly composed of preamble and body content (which may be split across multiple files for easier organisation on large documents).
The preamble of a document allows you to define header, footer, and section styles, colour palettes, and any other holistic document formatting easily, so revision-specific changes are easy to implement, signpost, and revert.
Everything can be programmatically defined with commands and counters in the preamble. 
For example, all section, subsection, table, and figure labels are customisable and referenced dynamically with \verb|\ref{item:<item_name>}|
\protect\footnote{We \verb|\label| each one, so \verb|\label{sec:introduction}| is referenced with \verb|\ref{sec:introduction}|.}
so we never need to go through an entire document and reindex if, say, a section is split into two or a new figure is added. 
Tables of contents are also dynamic because of this. 
Labels can be defined for anything, so dynamically referencing anything specific (like requirements) is possible, too. 
Citations are dynamically imported from a bibliography file, so something like changing referencing style is equally trivial. 
All of this makes changing larger projects easy, because the tedious work is automatically reevaluated.
\\
Tex typesetting is also known for its maths:
\begin{equation*}
  \int_{a}^{b} f(x) \d{x} = F(b) - F(a)
\end{equation*}
Which is well deserved. We can have inline, too: for $z \in \mathbb{C}, z = x+iy \; \forall \; x, y \in \mathbb{R}$:
\begin{align*}
  \sinh z &= \sum_{n=0}^{\infty} \frac{z^{2n+1}}{(2n+1)!} \\
          &= z + \frac{z^3}{3!} + \frac{z^5}{5!} + ...
\end{align*}
Finally, everything is vectorised so can be scaled to any resolution. This is handy for graphics.

\subsection{Theming}

Since everything is centrally defined in the preamble, we can set certain styles that can be tweaked locally and applied globally. Some good examples of this are section numbering styles:
\begin{minted}[baselinestretch=1.2]{latex}
\renewcommand{\thesection}{\Roman{section}.}
\end{minted}
Which would section numbers to be uppercase roman numerals,
custom caption styles:
\begin{minted}[baselinestretch=1.2]{latex}
\usepackage{caption}
\DeclareCaptionFormat{custom}{ 
  \textsc{#1#2} \\ #3 
}
\captionsetup{
  format=custom,
  font={it, stretch=1.2},
  labelsep=none
}
\end{minted}
Which would set captions to be the environment type\footnote{That is, a figure, image, table, etc. This is inferred by the compiler automatically.} in \textsc{Small Caps}, the number, and then a newline with the caption contents in italic. We can also set global palettes:
\begin{minted}[baselinestretch=1.2]{latex}
\definecolor{red}{rgb}{1, 0.2, 0.2}
\definecolor{green}{rgb}{0.2, 0.7, 0.2}
\definecolor{blue}{rgb}{0.2, 0.2, 1}
\end{minted}
Which can \textcolor{red}{then be used} \textcolor{green}{throughout} the \textcolor{blue}{document}.


\section{Graphics}
Packages like \texttt{tikzpicture} and \texttt{pgfplots} allow for dynamic, vectorised illustrations. Together they give a coordinate space that can be drawn on, and axes environments for plotting and analysing functions.

\subsection{Tikz}

Tikz is the main graphics package for creating diagrams, and gives an environment similar to Python's \texttt{turtle}. We draw on a grid, and also have access to fancy shapes.
Here's a simple Tikz example with code:

\begin{minted}[baselinestretch=1.2]{latex}
\begin{figure}[h] % makes a box on the page

    \begin{tikzpicture} % start a drawing

        \draw[fill=black] (0,0) circle (0.05);
        \draw (2,0) circle (1);
        \draw[fill=black] (2,0) circle (0.05);
        \draw[dashed] (0,0)--(2,0);

\subsection{Images}

Including images in Latex is straightforward, and we can use \texttt{tikz} in conjuction with images to annotate graphics easily and dynamically. We can include an image with the \verb|\includegraphics{path/to/image.png}| command, where the image directory is specified in the preamble:
\begin{figure}[htb!]
  \centering
  \includegraphics[width=0.6\textwidth]{model_rear.jpg}
  \label{img:plain}
  \caption{A Lego model of a BBC Micro with Archimedes CRT, from behind.}
\end{figure}
\newpage
Now, if we embed this in the \texttt{tikz} environment, we can annotate it.
\begin{figure}[htb!]
  \centering
  \begin{tikzpicture}
    \node[anchor=south west,inner sep=0] (image) at (0,0) {\includegraphics[width=0.6\textwidth]{model_rear.jpg}};
    \begin{scope}[x={(image.south east)},y={(image.north west)}]
      \draw[red,ultra thick,rounded corners, rotate around={15:(0.6,0.375)}] (0.45,0.25) rectangle (0.75,0.5);
      \node (a) [red, fill=white, rounded corners] at (0.9, 0.1) {Input and output};
      \draw[white, thick, ->] (a.north west)--(0.65, 0.25);

      \draw[red,ultra thick,rounded corners] (0.1,0.23) rectangle (0.3,0.45);
      \node (b) [red, fill=white, rounded corners] at (-0.2, 0.34) {Mouse (wireless)};
      \draw[white, thick, ->] (b.east)--(0.09, 0.34);
    \end{scope}
  \end{tikzpicture}
  \label{fig:annotated_image}
  \caption{The same model, annotated.}
\end{figure}

These can now be dynamically scaled, and all of the annotations will scale with it, while retaining control over their contents. 
This is superior to externally created or rasterised graphics, as all changes can be made within the \texttt{.tex} file.

\section{Commands}

Latex also provides programmatic ways of working with documents. The most useful aspect of this is defining specific macros, or \texttt{commands}, in the preamble. These let you program functions that are needed for a document, and can be called wherever they have been defined to be valid (ie. the body, footnotes, figures). As an example, we can define a command to draw, inside the Tikz environment, a cylinder with some specified dimensions and rotation:

\begin{minted}[baselinestretch=1.2, breaklines]{latex}
\newcommand{\cylinder}[3]{ % \cylinder takes three arguments
  \def\radius{#1} % take radius
  \def\height{#2} % take height
  \def\rotation{#3} % take rotation
  \def\eccentricity{0.5} % eccentricity of circular 

  % sides 
  \draw[rotate=\rotation] (-\radius, -\height/2)--(-\radius, \height/2);
  \draw[rotate=\rotation] (\radius, -\height/2)--(\radius, \height/2);

  % circular faces 
  \draw[rotate=\rotation] (0,\height/2) ellipse [x radius = \radius, y radius = \eccentricity*\radius];
  \draw[dashed, rotate=\rotation] (\radius, -\height/2) arc (0:180:{\radius} and {\eccentricity*\radius});
  \draw[rotate=\rotation] (-\radius, -\height/2) arc (180:360:{\radius} and {\eccentricity*\radius});
}
\end{minted}
We can now call this in the \texttt{tikzpicture} environment with different passed values, so \verb|\cylinder{2}{1}{0}| draws a cylinder of radius 2, height 1, and no rotation\footnote{Tikz has a standard unit for the increments on the grid, but we can also use centimeters, inches, em, points, or pixels instead.}:
\begin{center}
  \begin{tikzpicture}
    \cylinder{2}{1}{0}
  \end{tikzpicture}
\end{center}
And \verb|\cylinder{0.5}{3}{60}| gives
\begin{center}
  \begin{tikzpicture}
    \cylinder{0.5}{3}{60}
  \end{tikzpicture}
\end{center}
We also have access to flow control, iteration, and binary conditionals, so we could (if we wanted to) add an \texttt{origin} argument to the function, and iterate over some $x$ and $y$ values to draw cylinders with different rotations iff $x$ and $y$ are both prime. This can be quite powerful.
